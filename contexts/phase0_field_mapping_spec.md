# Phase 0 Field Mapping Specification
# Transformed Tables for New DB Design

## 1. Purpose

This document defines field-level mapping rules for transformed tables so migrations and backfill scripts can be implemented consistently and safely.

Conventions:

- source = current backend table/model field
- target = new design field
- rule = mapping and transformation rule
- null/default = target behavior when source is absent

## 11. Data Quality Gates Before Cutover

1. No orphan foreign keys after mapping
2. No duplicate business keys in target uniques
3. No nulls in required fields post-backfill
4. Inventory totals validated against baseline
5. Financial totals (daily/weekly) matched across old and new
6. Role assignment completeness = 100 percent for active users
